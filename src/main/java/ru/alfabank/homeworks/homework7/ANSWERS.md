1. Почему heroesCreated должен быть статическим? Потому что если его сделать нестатическим, значения счётчика можно будет подменять из любого другого класса и он перестанет правильно считать количество экземпляров всех героев
2. Почему name, health, level не могут быть статическими? Потому что с ними работают методы, которые непосредственно задают значения этих полей. Если сделать их статическими, это станет невозможно делать.
3. Почему printHeroesCreated() можно вызвать без создания объекта? Потому что данный метод оперирует как раз-таки статической переменной, которую больше никто не может изменить.
4. Можно ли из статического метода напрямую обратиться к нестатическому полю name? Почему? Нельзя, потому что при вызове из статического метода нестатическое поле ещё не инициализировано и не имеет никакого значения.
5. Почему проект перестанет компилироваться если пометить класс Hero модификатором final? Потому что при таком модификаторе доступа невозможно создавать классы-наследники.
6. Почему переменные имеют тип Hero, но при этом вызываются версии метода attack() из конкретных наследников? Потому что при добавлении в массив этих переменных они инициализировались как переменные классов наследников.
7. Почему переменная knight имеет модификатор final, но ее поля могут быть изменены? Потому что это не примитивная переменная, а экземпляр класса, поля которого можно менять с помощью геттеров и сеттеров.
